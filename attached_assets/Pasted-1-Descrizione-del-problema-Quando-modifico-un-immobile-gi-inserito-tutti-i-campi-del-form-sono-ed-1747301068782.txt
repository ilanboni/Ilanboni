1. Descrizione del problema
Quando modifico un immobile già inserito, tutti i campi del form sono editabili tranne i sei campi relativi alle agenzie (Agenzia 1, Agenzia 2, Agenzia 3 e Link Agenzia 1, Link Agenzia 2, Link Agenzia 3): non è possibile digitare nulla al loro interno.

2. Comportamento atteso
Durante la modifica, tutti i campi devono poter essere aggiornati con nuovi valori.

3. Comportamento effettivo
Gli input delle agenzie risultano “bloccati” (pur non essendo disabilitati via HTML), non ricevono eventi di digitazione né modifiche.

4. Passi per riprodurre il problema

Aprire l’app su Replit all’indirizzo <tuo-url-replit>

Inserire un nuovo immobile compilando tutti i campi, comprese Agenzia 1/Link Agenzia 1 ecc.

Salvare l’immobile (viene registrato correttamente).

Cliccare su “Modifica” di quell’immobile.

Provare a digitare nei campi “Agenzia 1/2/3” o “Link Agenzia 1/2/3”: il cursore appare ma la tastiera non immette testo.

5. Codice rilevante
(Incolla qui il frammento di HTML/JSX e lo snippet di JavaScript che gestisce il binding del form. Ad esempio:)

html
Copia
Modifica
<!-- input per Agenzia 1 -->
<input 
  type="text" 
  id="agency1" 
  name="agency1"

  onChange={this.handleChange} 
/>
js
Copia
Modifica
handleChange(e) {
  const { name, value } = e.target;
  this.setState({ [name]: value });
}
